# Translations

Zoo ships in **eight languages**. English is the source locale; the rest carry real, voice-preserving
translations (see the `zoo-i18n` skill — translate the *feeling*, not the words).

| Locale | Language              | File                        |
|--------|-----------------------|-----------------------------|
| en     | English               | `harbour-zoo-en.ts`         |
| fr     | French                | `harbour-zoo-fr.ts`         |
| de     | German                | `harbour-zoo-de.ts`         |
| it     | Italian               | `harbour-zoo-it.ts`         |
| es     | Spanish               | `harbour-zoo-es.ts`         |
| fi     | Finnish               | `harbour-zoo-fi.ts`         |
| zh_CN  | Chinese (Simplified)  | `harbour-zoo-zh_CN.ts`      |
| zh_TW  | Chinese (Traditional) | `harbour-zoo-zh_TW.ts`      |

The `.ts` files are generated by `lupdate` from `qsTr()`/`tr()` strings once the QML/C++ exists
(they don't exist yet — there are no source strings to extract). Regenerate after adding any
user-visible string:

```bash
lupdate harbour-zoo.pro       # extract/update source strings into every .ts
lrelease translations/*.ts    # compile to .qm (also automatic via CONFIG += sailfishapp_i18n)
```

Add the `TRANSLATIONS +=` block (all eight files) and `CONFIG += sailfishapp_i18n` to
`harbour-zoo.pro` when scaffolding the project. See the `zoo-i18n` skill for the full workflow and
the rule that the Keeper's warm, absurd voice must survive into every language.
